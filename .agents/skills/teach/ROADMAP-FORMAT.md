# ROADMAP.md Format

`ROADMAP.md` is the compact curriculum for this workspace. It maps the mission to the next concepts, repo areas, and milestones worth learning. It is not a full project plan; it is a teaching compass.

## Template

```md
# {Topic} Roadmap

## How to use this roadmap
{1-2 sentences on how lessons should be chosen from this roadmap.}

## Current focus
- {The concept or milestone to prioritize next}

## Learning tracks

### {Track name}
- Why it matters: {why this track matters for the mission}
- Repo anchors: `{path}`, `{path}`, `{path}`
- Likely lessons:
  - {Short lesson idea}
  - {Short lesson idea}
- Proof signals:
  - {test, eval, code path, write-up, or demo that proves learning}

### {Another track}
- Why it matters: {…}
- Repo anchors: `{path}`, `{path}`
- Likely lessons:
  - {…}
- Proof signals:
  - {…}
```

## Rules

- **Keep it short.** This is a roadmap, not a full syllabus.
- **Tie each track to repo evidence.** A track without concrete files, tests, or milestones is probably too abstract.
- **Update when priorities move.** The roadmap should reflect the next useful thing to learn, not a stale backlog.
- **Bias toward shipping.** Prefer tracks that help the user understand and complete the next real project step.
