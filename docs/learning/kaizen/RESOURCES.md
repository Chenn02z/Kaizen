# Kaizen Learning Resources

## Local project sources

- `docs/PRODUCT.md`
  Product intent and success criteria. Use for: understanding what the system should optimize for and what counts as a meaningful capability.
- `docs/milestones/`
  Milestone acceptance criteria and scope boundaries. Use for: deciding what to learn next and what to ignore for now.
- `AGENTS.md`
  Repo rules and definition of done. Use for: how work should be executed and verified in this codebase.
- `app/`
  Concrete implementation examples. Use for: turning abstract AI engineering ideas into code-level understanding.
- `tests/` and `evals/`
  Evidence and guardrails. Use for: learning how behavior is checked instead of trusted.
